AULA 02 => NÍVEIS E TÉCNICAS DE TESTES

NÍVEIS DE TESTES: UNIDADE -> INTEGRAÇÃO -> SISTEMA -> REGRESSÃO -> ACEITAÇÃO

1° TESTE - *UNIDADE* : SE PREOCUPA COM A PEQUENA PARTE DO CÓDIGO (JUNIT , PHPUNIT, JEST)

2° TESTE - *INTEGRAÇÃO* : JUNTAR ESSA PEQUENA PARTE A UM OUTRO CONJUNTO (INTEGRATION)

3° TESTE - *SISTEMA*: ABRIR O APLICACTIVO, SOFTARE PARA O USUÁRIO 

4° TESTE - *REGRESSÃO*: MANTER O SISTEMA OK, ENQUANTO LANÇA ATUALIZAÇÕES (SELENNIUM).

--- ALPHA , BETA , CANNARY ---

ALPHA É O PRIMEIRO TESTE, NORMALMENTE O MAIS SIMPLES PORQUE É O PRIMEIRO "SEMPRE HAVERÁ ALGUM ERRO PARA CORRIGIR"

BETA É PRA UMA GAMA MAIOR DE USUARIOS, FAZER UMA VALIUDAÇÃO MAIOR EM GRUPO

CANNARY ESTRATÉGIA DE DEPLOY, IMPLANTAR O SISTEMA EM PRODUÇÃO

5° TESTE - *ACEITAÇÃO*: USUÁRIO FAZ OS TESTES E TIRA CONCLUÇÃO SE VAI QUERER FICAR COM O SISTEMA

- - - | | - - -

TÉCNICAS DE TESTES

CAIXA BRANCA, ESTRUTURAL -> CONHECER O CÓDIGO FONTE, SABER COMO FUNCIONA EMBAIXO DOS PANOS (UNIDADE, INTEGRAÇÃO, REGRESSÃO)

CAIXA PRETA, FUNCIONAL -> NÃO CONHECER O QUE ACONTECE, (INTEGRAÇÃO, SISTEMA, ACEITAÇÃO)

CAIXA CINZA -> MESCLA DAS DUAS TÉCNICAS ACIMA, ENGENHARIA REVERSA
